使用指南

1. 提交分析

  • 在左侧上传区拖拽或点击上传 .json 或 .json.gz 文件（PyTorch Profiler 导出的 Chrome Trace 格式）
  • 选择所属项目（可选，用于分类管理）
  • kernel-types 留空则按规则自动分类（时长占比 >8%、数量占比 >10% 等）；也可填入额外关键词，逗号分隔
  • 点击"提交分析"，等待完成

2. 查看结果

  • 控制台：原始文本输出
  • 图表：Kernel 类型耗时柱状图和占比饼图
  • CSV 表格：支持搜索、列筛选、排序、列宽拖拽调整

3. 项目管理

  • 点击"+ 新建项目"创建项目，用于分类管理分析任务
  • 选择历史记录后可移动到其他项目或删除

4. 历史对比

  • 在左侧"对比"标签页选择两个已完成的单文件任务
  • 可选填 kernel-types 额外关键词和备注，发起对比分析（Kernel 类型自动分类）
  • 无需重新上传文件，直接复用已有数据

5. 其他功能

  • Perfetto：点击按钮在 Perfetto UI 中打开 trace 文件
  • 下载 CSV：在表格右上角可下载当前视图数据
  • 查看代码：Triton 表格中点击"查看代码"可查看生成的 kernel 代码

6. 运行 Triton 代码

  • 在 Triton 或 Triton Step N 表格中，每行有"运行"按钮
  • 点击运行后会执行对应的 Triton kernel 代码，弹出执行结果
  • 效率值会显示在行内（单位 GB/s），旁边有 ↻ 按钮可重新运行
  • 清除 Cache：在 Triton Step N 表格右上角有"清除 Cache"按钮，可清除 /tmp/torchinductor_* 缓存目录，重新运行以获得准确性能数据

注意：此功能依赖 MLU（寒武纪）硬件环境。需要已安装 torch_mlu 驱动并有物理 MLU 加速卡。 在非 MLU 环境下执行会报 ImportError，这是正常现象，不代表程序出错。
